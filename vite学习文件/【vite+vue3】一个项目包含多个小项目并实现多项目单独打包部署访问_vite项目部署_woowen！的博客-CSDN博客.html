公司有两个项目所以想着在一个大的项目里实现两个小项目:projectA和projectB项目,公共部分提取出来.

例如项目文件结构如下：

  1. ├── package.json
  2. ├── vite.config.js
  3. ├── index.html // 可以删除

  6. └── src
  7.     ├── main.ts // 可以删除
  8.     ├── App.vue //可以删除

  11.     └── components // 公用组件
  12.     └── project
  13.         └── projectA
  14.             └── utils // 各自的具包

  19.             ｜————main.ts
  20.             ｜————App.vue

  23.             ｜————api 
  24.             ｜————router

  26.             ｜————index.html

  28.             ｜————App.vue

  30.     └── assets // 静态文件
  31.     └── store  // 状态容器

主要实现要点：

1.  projectA项目和projectB项目中均要保留index.html、main.ts/js 、App.vue 这三个文件可以直接复制粘贴src下的index.html、main.ts/js 、App.vue文件内容，需要注意的是main.ts路径引入的变化,以及路由的路径引入正确与否

例如

 2. 配置vite.config.ts 2.快速 配置 . ts

  1. import vue from '@vitejs/plugin-vue'
  2. import path from 'path'
  3.  
  4. export default defineConfig({
  5.    // 项目1
  6.     projectA: {
  7.       // 入口文件
  8.       entry: path.resolve(__dirname, 'src/projects/projectA/main.js'),
  9.       // 打包输出路径
  10.       outDir: path.resolve(__dirname, 'dist/projectA'),
  11.       // 生成的 HTML 文件路径
  12.       html: {
  13.         filename: 'index.html',
  14.         title: 'Project 1',
  15.       }
  16.     },
  17.     // 项目2
  18.     projectB: {
  19.       entry: path.resolve(__dirname, 'src/projects/projectB/main.js'),
  20.       outDir: path.resolve(__dirname, 'dist/projectB'),
  21.       html: {
  22.         filename: 'index.html',
  23.         title: 'Project 2',
  24.       }
  25.     }
  26.   }
  27.   
  28.   build: {
  29.     target: 'modules',
  30.     // 实现打包文件放在dist路径的 env.VITE_APP_NAME下， env.VITE_APP_NAME为环境变量，看下面的打包命令
  31.     outDir: 'dist/'+ env.VITE_APP_NAME + '/', 
  32.     assetsDir: 'static', //  这里是打包后文件的存放路径
  33.     sourcemap: true,
  34.     rollupOptions: {
  35.       output: {
  36.           // vite打包以后的js,css和img资源分别分门别类在js/css/img文件夹中
  37.           entryFileNames: 'static/js/[name]-[hash].js',
  38.           chunkFileNames: 'static/js/[name]-[hash].js',
  39.           assetFileNames: 'static/[ext]/name-[hash].[ext]',
  40.       }
  41.     }
  42.   },
  43.   server: {
  44.     port: 3000,
  45.     open: true,
  46.     proxy: {}
  47.   },
  48.   resolve: {
  49.     alias: {
  50.       '@': path.resolve(__dirname, 'src')
  51.     }
  52.   }
  53.     
  54. })

3. 配置package.json3.配置package.json

  1. "scripts": {
  2.     "i": "pnpm install",
  3.     "dev:A": "vite --mode projectA",
  4.     "dev:B": "vite --mode projectB",
  5.     "build:A": "node --max_old_space_size=8000 ./node_modules/vite/bin/vite.js build --mode projectA",
  6.     "build:B": "node --max_old_space_size=8000 ./node_modules/vite/bin/vite.js build --mode projectB",
  7.  
  8. }
  9. // --mode projectA  指的是使用env.projectA 文件中的变量

4. 环境变量的配置

env.porjectA文件：

  1. # 标题
  2. VITE_APP_TITLE=我是项目标题A
  3.  
  4. # 子项目名称
  5. VITE_APP_NAME=portal

env.porjectB文件：

  1. # 标题
  2. VITE_APP_TITLE=我是项目标题B
  3.  
  4. # 子项目名称
  5. VITE_APP_NAME=risk
